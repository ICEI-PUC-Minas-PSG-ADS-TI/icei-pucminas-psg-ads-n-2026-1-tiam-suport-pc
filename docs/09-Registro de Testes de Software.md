#  Registro de Testes de Software

## 1. Visão Geral
Este documento apresenta os resultados dos testes realizados sobre o protótipo da aplicação, desenvolvido na ferramenta **Figma**.

Os testes tiveram como objetivo avaliar a **usabilidade da interface**, a **clareza das informações** e a **compreensão do fluxo de navegação** pelos usuários.

---

## 2. Resultados dos Testes
Os testes foram conduzidos com base em cenários simulados, nos quais os participantes interagiram com o protótipo navegável do sistema.

> [!IMPORTANT]
> De forma geral, os usuários compreenderam o funcionamento da aplicação. No entanto, foram identificados pontos de melhoria na organização das informações e no fluxo de navegação.

---

## 3. Pontos Fortes 
Durante os testes, os seguintes aspectos positivos se destacaram:

*   **Interface:** Simples e de fácil compreensão.
*   **Navegação:** Intuitiva entre as telas principais.
*   **Funcionalidades:** Facilidade na identificação dos recursos.
*   **Estrutura:** Fluxo geral do sistema bem estruturado.
*   **Clareza:** Apresentação nítida dos serviços disponíveis.

---

## 4. Pontos Fracos 
Dificuldades observadas durante a interação com o protótipo:

*   Dúvidas iniciais sobre o fluxo de agendamento.
*   Falta de destaque em botões de ação prioritários.
*   Necessidade de melhor organização de dados em telas específicas.
*   Ausência de indicações visuais de orientação (direcionamento).

---

## 5. Problemas Identificados 
Erros críticos ou barreiras de usabilidade:

1.  **Indecisão no Fluxo:** Dificuldade em identificar o próximo passo após selecionar um serviço.
2.  **Falta de Feedback:** Ausência de resposta visual em ações importantes (ex: confirmação de agendamento).
3.  **Tentativa e Erro:** Navegação pouco evidente em algumas telas, exigindo cliques aleatórios.

---

## 6. Melhorias Propostas 
Com base nos resultados, as seguintes ações foram definidas:

| Categoria | Ação Corretiva |
| :--- | :--- |
| **Feedback** | Inserção de alertas e sinais visuais para ações do usuário. |
| **Interface** | Destaque visual nos botões principais (Call to Action). |
| **Conteúdo** | Reorganização das informações na tela de agendamento. |
| **UX Design** | Inclusão de elementos visuais que orientem o fluxo de uso. |
| **Hierarquia** | Ajustes na priorização visual das informações em tela. |

---

## 7. Considerações Finais
Os testes demonstraram que o protótipo **atende aos objetivos propostos**, permitindo a compreensão geral da aplicação. As melhorias identificadas serão aplicadas nas próximas etapas do projeto para garantir uma experiência de usuário superior.
